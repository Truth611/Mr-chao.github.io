2019年中总结

发表于 2019-07-13 | 分类于 思考 | 访问数 °C
字数统计 1,156 | 阅读时长 4

2019 年上半年

1月份干了什么还真忘了，可能在学校准备物联网比赛、期末作业、准备期末考试吧。

2月 极客时间 大概了解了 tensorflow，以及一点数据可视化。这个时候是寒假，在家也就干了这个事儿。

3月 要为4月 5月的比赛有所准备了。

4月的 robocup 中国赛比赛很顺利，灵感来了啪啪啪就把代码敲出来了，这个比赛对我带来的好处：
1、阅读大量源码的经验，以后不怂了！来吧，干~
2、一点 C++ 语法，了解了一点 C++，会运用了并取得了好的结果，这事啃书本得不到的经验与体会，想要深入 C++ 还得深入书本，做做项目。
3、lua 脚本语言接触了解、类似于 python，然后后面了解了 Openresty，学起来也得心应手一些，以及目前7月份接触到了 Ruby 语言，也没那么怂了，脚本语言嘛，思想在就不怕，学起来很快的。
4、做这个比赛最重要的是团结、团队的沟通，赛前我也在一直努力传达我的思想，争取我的代码不会与其他人的冲突，这份经验难得可贵。
5、解决问题的经验、思想，很难得，赛前赛后会碰到各种各样的情况，太复杂了，感谢 BUG 让我~

5月的 安徽省物联网竞赛——技能组：

谈谈这个比赛：

3月份准备一部分，4月也准备了一部分，毕竟中间有个机器人比赛，我自己也不想在这个国赛中留下遗憾，所以中心偏在国赛上。

3月初，大家的精力还在考研复习上，所以也就利用晚上的时间敲敲国赛代码。我呢看心情哈哈~两个比赛一起准备。

泡了一两个星期的图书馆，把 Flask 框架熟悉了一遍，这个框架真的牛批唉~Django 之前就有了解，大二下 2 个课程设计就是自学用了 Django 框架，所以 Flask 学起来也是挺快的，后来呢有时间又做了小站，算是把 Django 框架再熟悉一遍。

比赛时要用到小程序，做一个智能植物保姆，核心在于物联网网关的设计，多端使用。后端采用 Django，前后端通信不熟悉，所以想要直接敲出比赛代码，特别时后端和小程序通信部分很难。

4月在之前学习的基础上，通宵学习做这个小程序，我觉得厚积薄发很重要，没有之前的积累我是不可能在短时间内完成小程序方面的东西。

比赛结果：
机器人国赛：二等奖 ps：晋级的时候碰到世界杯冠军，我们差点把他们淘汰了~换个裁判真给他淘汰掉！
物联网比赛：三等奖 口才不照，答辩都没把东西表达出来，队友又在准备考研，也没好让他们多准备。 ps:让我拉来比赛的朋友二等，以一挡百的实例，搞的机器学习，中间我也稍微了解 OB 一下了，哈哈~
机器人省赛：二等奖 ps：毕设免答辩了

5月比赛结束之后：
21 天的机器学习之路，了解了基本的一些名词，简单的原理。通过这个让我更加认清了数学的重要性。

6月启航，开始泡图书馆，学数学，开始有考研的打算。
学数学；接触到了 openresty；过了一遍 Linux 和 shell；学了一点 web 网络协议相关的东西（有点深的感觉，不如刷书）

7月 定计划，定目标，学习。
又买了几本书，《统计学习方法》《程序员数学系类 3 本》，打算学数学嘛，应用才是目的，如果我离开学校了，一定再也没机会这么认真学习数学了，不甘心，大一大二欠的数学债慢慢还。

2019 下半年

《统计学习方法》《程序员数学系类 3 本》然后再买本李沫的《动手学深度学习》搭配着搞一搞深度学习的原理。有目的的学数学。

数据分析方面《python数据科学手册》《python绝技》
试一试搞明白聚类分析什么的底层原理。

代码都是现成的，拿来用就是，用是用了，不会底层原理又有什么用呢，永远搬砖吗？

上岸~
加油~
